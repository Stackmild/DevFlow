## 4. 状态系统

| 状态 | 用户能否继续操作 | 系统显示什么 | 用户下一步 | 是否保留上下文 |
|------|--------------|-----------|---------|------------|
| default | ✅ | 正常内容 | - | - |
| loading | ❌ | skeleton/spinner | 等待 | ✅ |
| empty | ✅ | 空态插图+文字 | 创建/导入 | - |
| error | ✅ | 错误提示+重试 | 重试/反馈 | ✅ |
| disabled | ❌ | 灰色+说明 | 查看原因 | - |
| sync pending | ✅ | 同步中提示 | 等待/刷新 | ✅ |
| not ready | ✅ | 数据未准备好提示 | 等待 | - |
| permission denied | ✅ | 无权限说明 | 申请权限/联系管理员 | - |

## 7. Persona 差异与数据前提

### Persona 清单

| Persona | 特征 | 关键差异 | 需要特殊处理 |
|---------|------|---------|----------|
| Owner/管理员 | 全量数据视角 | - | - |
| 普通业务用户 | 项目子集有限 | 列表/聚合结果不同 | 需提示为什么看不到数据 |
| 弱数据用户（新同步） | 字段缺失概率高 | 大量 fallback 状态 | 数据准备中的明确提示 |

### 数据前提依赖

| 依赖类型 | 若未完成的用户可见表现 | 处理方式 |
|---------|-------------------|---------|
| sync | 列表为空或数据陈旧 | 显示"同步中"状态 |
| init-state | Dashboard 统计为零 | 显示"初始化中"状态 |
| backfill | 关键字段显示"-" | 合理 fallback + 说明 |
